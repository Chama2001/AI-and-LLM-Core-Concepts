# 🗂️ Vector Databases (AI සඳහා දත්ත ගබඩා සංකල්ප)

සාමාන්‍යයෙන් අපිට හුරුපුරුදු MySQL, PostgreSQL වැනි Relational Databases වල දත්ත ගබඩා කරන්නේ Tables, Rows සහ Columns විදිහටයි. ඒවායේ Search කරන්නේ නිශ්චිත වචනයක් (Exact Keyword) පදනම් කරගෙනයි.

නමුත් AI සහ Large Language Models (LLMs) සමඟ වැඩ කිරීමේදී අපිට වචන වල **"තේරුම" (Semantic Meaning)** මත පදනම්ව දත්ත සෙවීමට සිදු වෙනවා. ඒ සඳහා පාවිච්චි කරන විශේෂිත දත්ත ගබඩාවන් තමයි **Vector Databases** කියන්නේ.

---

## 🧠 1. සාමාන්‍ය DB සහ Vector DB අතර වෙනස

* **සාමාන්‍ය DB (MySQL):** ඔයා "කාර්" (Car) කියලා search කළොත්, ඒ වචනය තියෙන දේවල් විතරක් පෙන්වයි. "වාහන" (Vehicle) හෝ "මෝටර් රථ" කියන වචන තියෙන ඩේටා මඟහැරෙන්න පුළුවන්.
* **Vector DB (Pinecone / ChromaDB):** ඔයා "කාර්" කියලා search කළොත්, ඒ වචනය නැතත් ඒකට සමාන තේරුමක් තියෙන "වාහන", "රෝද හතරේ ඒවා", "Automobile" වැනි ඕනෑම සමාන තේරුමක් තියෙන දත්ත හොයාගන්න පුළුවන්.

---

## 🛠️ Vector Database එකක් වැඩ කරන්නේ කොහොමද?

1. **Embeddings:** මුලින්ම අපි දෙන Text (Document කැබලි) **Embedding Model** එකක් හරහා යවලා, ඒ වචන වල තේරුම නිරූපණය වන පරිදි ගණිතමය අංක සමූහයක් (Numbers/Vectors) බවට පත් කරනවා.
2. **High-Dimensional Space:** මේ Vectors (ඛණ්ඩාංක) ගණිතමය අවකාශයක ලකුණු කරනවා. සමාන තේරුම් තියෙන වචන (උදා: 'පරිගණකය' සහ 'ලැප්ටොප්') එක ළඟ පිහිටන විදිහටත්, වෙනස් තේරුම් තියෙන වචන (උදා: 'ඇපල් ගෙඩිය' සහ 'මෝටර් රථය') ඈතින් පිහිටන විදිහටත් මේ Vectors සකස් වෙනවා.
3. **Similarity Search (සමානකම් සෙවීම):** User කෙනෙක් ප්‍රශ්නයක් ඇහුවාම, ඒ ප්‍රශ්නයේ Vector එකට ඩේටාබේස් එක ඇතුළේ **ජ්‍යාමිතිකව (Geometrically) වඩාත්ම ළඟින්ම පිහිටන** Vectors මොනවාද කියලා ගණනය කරලා (Cosine Similarity වැනි ගණිත ක්‍රම මඟින්) අදාළම උත්තර ටික සොයා ගන්නවා.

## 🚀 ජනප්‍රිය Vector Databases මෙවලම් කිහිපයක්

* **Pinecone:** සම්පූර්ණයෙන්ම Cloud-based (Managed) ඉතාමත් ජනප්‍රිය එකක්.
* **ChromaDB:** Open-source සහ අපේ පරිගණකයේම (Locally) ලේසියෙන්ම රන් කරගන්න පුළුවන් එකක් (Python/Next.js projects වලට බහුලව පාවිච්චි කරයි).
* **Milvus / Qdrant:** විශාල දත්ත ප්‍රමාණයක් (Enterprise level) සමඟ වැඩ කිරීමට පාවිච්චි කරයි.
