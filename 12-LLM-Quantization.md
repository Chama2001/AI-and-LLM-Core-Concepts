# 📉 LLM Quantization & Model Compression (AI මොඩලයන්ගේ ප්‍රමාණය කුඩා කිරීම)

Large Language Models (LLMs) බිලියන ගණනක Parameters වලින් සමන්විත නිසා ඒවා ක්‍රියාත්මක කිරීමට (Inference) අධික මිල අධික GPUs සහ විශාල RAM ධාරිතාවක් අවශ්‍ය වේ. 

සාමාන්‍ය පරිගණකයක හෝ ජංගම දුරකථනයක (Locally) අඩු පහසුකම් යටතේ AI මොඩලයක් වේගයෙන් රන් කරවීම සඳහා, එහි ගුණාත්මකභාවය ලොකුවට හානි නොකර මොඩලයේ ප්‍රමාණය කුඩා කරන ප්‍රධානම තාක්ෂණය **Quantization** ලෙස හඳුන්වයි.

---

## 🧠 1. Quantization කියන්නේ සරලවම මොකක්ද?

AI මොඩලයක තියෙන හැම Parameter එකක්ම (Weight එකක්ම) ගබඩා වෙලා තියෙන්නේ ඉතාමත් සවිස්තරාත්මක දශම සංඛ්‍යා (Floating-point numbers) විදිහටයි.
* **FP32 (Full Precision):** සාමාන්‍යයෙන් මොඩලයක් Train කරද්දී හැම Weight එකක්ම බිටු 32ක (32-bit) දශම අගයක් ගනියි. (ඉතා නිවැරදියි, නමුත් ඉඩ ගොඩක් ගනියි).
* **Quantization ක්‍රියාවලිය:** මෙහිදී කරන්නේ ඒ 32-bit දශම සංඛ්‍යා, බිටු 8ක (8-bit) හෝ බිටු 4ක (4-bit) සරල පූර්ණ සංඛ්‍යා (Integers - INT8/INT4) බවට පරිවර්තනය කිරීමයි.

**💡 සරල උදාහරණයක්:** 
දශම 4ක් තියෙන `3.14159265` වැනි සංඛ්‍යාවක් සරලව `3.14` හෝ `3` ලෙස වටයා (Round-off) ලිවීම වැනි දෙයකි. මෙයින් මොඩලයේ ප්‍රමාණය (File size) 4 ගුණයකින් පමණ කුඩා කරගත හැක!

## 📊 Quantization වල වාසි (Advantages)

1. **අඩු Memory (RAM) භාවිතය:** උදාහරණයක් ලෙස 70 Billion parameter තියෙන මොඩලයක් FP32 වලින් රන් කරන්න 140GB+ VRAM අවශ්‍ය වන අතර, එය 4-bit වලට Quantize කළ විට 40GB ක RAM එකකින් රන් කළ හැක.
2. **ඉහළ වේගය (Faster Inference):** 32-bit දශම ගණන් හදනවාට වඩා 4-bit පූර්ණ සංඛ්‍යා (Integers) ගණනය කිරීම ප්‍රොසෙසරයකට ඉතාමත් වේගවත්ය. එබැවින් AI එක පිළිතුරු දෙන වේගය (Tokens per second) වැඩිවේ.
3. **Local Deployment:** AI මොඩලයක් Cloud එකට නොයවා, සේවාදායකයෙකුගේ (Client) පරිගණකයක් තුළම Offline රන් කරවීමට (Edge Computing) මඟ පාදයි.

## 🛠️ ජනප්‍රිය Quantization Formats වර්ග

ඔයා Hugging Face වැනි වෙබ් අඩවි වලින් මොඩලයක් ඩවුන්ලෝඩ් කරද්දී මේ නම් දකින්න ලැබෙනවා:

* **GGUF (GPT-Generated Unified Format):** CPU එකක් සහ සාමාන්‍ය RAM එකක් තියෙන ඕනෑම ලැප්ටොප් එකක AI මොඩලයක් රන් කරන්න හදපු හොඳම Format එකයි (Ollama, LM Studio මෙවලම් වල බහුලව පාවිච්චි වේ).
* **AWQ / GPTQ:** මේවා GPU (NVIDIA காரඩ්) මත වඩාත් වේගයෙන් සහ අවම ගුණාත්මක හානියකින් රන් කිරීමට සකස් කළ Formats වේ.
* **EXL2:** ඉතාමත් වේගවත් GPU inference සඳහා පාවිච්චි කරයි.
