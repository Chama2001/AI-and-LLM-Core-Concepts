# 🧠 Context Window & Token Limits (AI මොඩලයන්ගේ මතක සීමාව)

Large Language Model (LLM) එකකට ප්‍රශ්නයක් ඇසීමේදී, එයට එකවර කියවිය හැකි සහ මතකයේ තබාගත හැකි උපරිම Text (Tokens) ප්‍රමාණය සීමිතයි. මෙන්න මේ උපරිම සීමාවට තාක්ෂණිකව **Context Window** කියා කියනු ලබනවා.

අපි AI එකකට ලබාදෙන ප්‍රශ්නය (Prompt) සහ AI එක ආපසු ලබාදෙන පිළිතුර (Response) යන දෙකෙහිම එකතුව මේ Context Window එක ඇතුළත පැවතිය යුතුය.

---

## 📊 1. Context Window එක මනින්නේ කොහොමද?

Context Window එක මනින්නේ වචන ගණනින් හෝ Pages ගණනින් නෙවෙයි, **Tokens** ප්‍රමාණයෙනි.
* **උදාහරණයක්:** `GPT-4o` සතුව 128K (ಟෝකන 128,000) ක Context Window එකක් පවතී. මෙයින් අදහස් කරන්නේ එයට එකවර සාමාන්‍ය වචන 95,000ක් (පොතක පිටු 300ක් පමණ) එකපාර කියවා තේරුම් ගත හැකි බවයි.
* **Gemini 1.5 Pro:** සතුව මිලියන 2ක (2M Tokens) දැවැන්ත Context Window එකක් ඇත. එයට එකවර පැය ගණනක Video එකක්, මුළු Codebase එකක්ම, හෝ විශාල පොත් කිහිපයක් එකවර කියවීමේ හැකියාව ඇත.

---

## ⚠️ 2. Context Window එක ඉවර වුණාම වෙන්නේ මොකක්ද?

අපි ChatGPT හෝ වෙනත් AI එකක් සමඟ දිගු වේලාවක් Chat කරගෙන යද්දී (Long Conversation), කතාබහ කරන දේවල් ප්‍රමාණය Context Window එකට වඩා වැඩි වුවහොත්:
1. **ලැබෙන පිළිතුර කැපී යාම:** AI එක දෙන උත්තරය මැදින් නතර විය හැක.
2. **පරණ දේවල් අමතක වීම (Forgetfulness):** AI එක විසින් Chat එකේ මුලින්ම කතා කරපු කරුණු මතකයෙන් ඉවත් කර, අලුත්ම Tokens ටික පමණක් මතකයේ තබා ගනියි.

---

## 🛠️ Developers ලා මේක පාලනය කරන්නේ කොහොමද?

ලොකු Applications හදද්දී මේ ප්‍රශ්නය විසඳන්න ක්‍රම කිහිපයක් පාවිච්චි කරනවා:

* **Sliding Window:** Chat එකක් දිගටම යද්දී, හැමතිස්සෙම අවසානයට කතා කරපු Messages 5 හෝ 10 පමණක් LLM එකට මතක් කර දීම.
* **Conversation Summary:** පරණ Chat history එක සම්පූර්ණයෙන්ම යවන්නේ නැතිව, වෙනම AI එකක් ලවා ඒ පරණ දේවල් කෙටියෙන් සාරාංශ කර (Summary) LLM එකට ලබා දීම.
* **Max Tokens Parameter:** API එකකින් කෝඩ් කරද්දී `max_tokens` අගයක් ලබා දී AI එක උත්තර දීමට පාවිච්චි කළ යුතු උපරිම සීමාව පාලනය කිරීම (වියදම/Cost අඩු කරගැනීමට).
